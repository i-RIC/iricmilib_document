Creating “definition.xml”

definition.xml is the file that defines the interface of the model.

The grammer of definition.xml is the extended version of iRIC. Please refer to the URL below, about how to create definition.xml for iRIC.

https://iric-solver-dev-manual.readthedocs.io/en/latest/02/02_folder_solverdef.html#creating-a-solver-definition-file

There are two points that the model developer should be careful.

  • Defining time dependent input

  • Defining time dependent output

They are important, because they are the data that iRIC-MI framework will make it possible to exchange between models.

Defining time dependent input

There are two types of time dependent input:

  • Scalar value (calculation condition, boundary condition, complex type grid attribute)

  • Grid attribute

Please refer to Types of input and output for the detail about the types.

How to define time dependent input in definition.xml is described below.

Defining a scalar value

You can define scalar value time dependent input, as functional input. The key is that the caption of <Parameter> element should be “Time”.

When the caption is “Time”, the variable is recognized to be time dependent input value, and the value can be received using iRICMI_RIn_Real etc.

Listing 9 shows an example. It can appear in <CalculationCondition>, <BoundaryCondition/>, or <GridRelatedCondition type=”complex” />.

Listing 9 definition.xml to define single time dependent input value
1<Item name="discharge" caption="Discharge">
2  <Definition conditionType="functional">
3    <Parameter valueType="real" caption="Time"/>
4    <Value valueType="real" caption="Discharge(m3/s)"/>
5  </Definition>
6</Item>

Defnining a grid attribute

You can define time dependent grid attribute, as a grid attribute with dimension “Time”.

The key is that it contains the definition of <Dimension>, and the name of the <Dimension> is “Time”.

When a grid attribute satisfies the condition above, the grid attribute is recognized to be time dependent input data, and the values can be received using iRICMI_RIn_Grid_Node_Real etc.

Listing 10 shows an example.

Listing 10 definition.xml to define time dependent input value defined at grid cell
1<Item name="rain" caption="Rainfall">
2  <Definition position="cell" valueType="real" default="0">
3    <Dimension name="Time" caption="Time" valueType="real"/>
4  </Definition>
5</Item>

Defining time dependent output

Global

Listing 11 shows an example of global output.

Listing 11 definition.xml to define global output
 1<SolverDefinition
 2  xmlns="www.iric.net/SolverDefinition/1.0"
 3  name="sample"
 4  caption="Sample Model"
 5  version="1.0"
 6  copyright="iRIC organization"
 7  release="2020.6.20"
 8  homepage="http://i-ric.org"
 9  executable="sample.exe"
10  iterationtype="time"
11  gridtype="structured2d"
12>
13  <CalculationCondition>
14  </CalculationCondition>
15  <GridRelatedCondition>
16    <Item name="Elevation" caption="Elevation (m)">
17      <Definition position="node" valueType="real" default="max" />
18    </Item>
19  </GridRelatedCondition>
20  <GlobalOutput>
21    <Output name="discharge" caption="Discharge">
22      <Definition valueType="real" />
23    </Output>
24  </GlobalOutput>
25</SolverDefinition>

Boundary condition

Listing 12 shows an example of grid attribute output.

Listing 12 definition.xml to define boundary condition output
 1<SolverDefinition
 2  xmlns="www.iric.net/SolverDefinition/1.0"
 3      name="global_send_c++"
 4      caption="Model for testing sending global value (C++)"
 5      version="1.0"
 6      copyright="iRIC organization"
 7      release="2020.6.20"
 8      homepage="http://i-ric.org"
 9      executable="global_send_c++.exe"
10      iterationtype="time"
11      gridtype="structured2d"
12>
13  <CalculationCondition>
14  </CalculationCondition>
15  <BoundaryCondition name="inflow" caption="Inflow" position="edge">
16    <Item name="discharge" caption="Discharge">
17      <Definition valueType="real" default="0" />
18    </Item>
19    <Output name="water_elevation" caption="Water Surface Elevation">
20      <Definition valueType="real" />
21    </Output>
22  </BoundaryCondition>
23</SolverDefinition>

14  </CalculationCondition>

16    <Item name="area" caption="Area">
17      <Definition position="node" valueType="complex">

19          <Definition valueType="real" />
20        </Output>
21      </Definition>
22    </Item>
23  </GridRelatedCondition>
24</SolverDefinition>

Grid attribute

Listing 14 shows an example of grid attribute output.

For “position” attriute, use can use the values below:

  • node

  • cell

  • iface

  • jface

  • kface

Listing 14 definition.xml to define grid attribute output
 1<SolverDefinition
 2  xmlns="www.iric.net/SolverDefinition/1.0"
 3  name="sample"
 4  caption="Sample Model"
 5  version="1.0"
 6  copyright="iRIC organization"
 7  release="2020.6.20"
 8  homepage="http://i-ric.org"
 9  executable="sample.exe"
10  iterationtype="time"
11  gridtype="structured2d"
12>
13  <CalculationCondition>
14  </CalculationCondition>
15  <GridRelatedCondition>
16    <Item name="Elevation" caption="Elevation (m)">
17      <Definition position="node" valueType="real" default="max" />
18    </Item>
19    <Output name="Depth_node" caption="Depth (m)">
20      <Definition position="node" valueType="real"/>
21    </Output>
22    <Output name="Depth_cell" caption="Depth (m)">
23      <Definition position="cell" valueType="real"/>
24    </Output>
25  </GridRelatedCondition>
26</SolverDefinition>
