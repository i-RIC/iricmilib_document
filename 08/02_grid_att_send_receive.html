grid_att_send and grid_att_receive

grid_att_send and grid_att_receive are the models that send and receive time dependent grid attribute value “rain”.

grid_att_send and grid_att_receive are implemented in FORTRAN, C++ and Python. On this page FORTRAN version is explained.

grid_att_send

Listing 29 shows “definition.xml” of grid_att_send, and Listing 30 shows the source code of grid_att_send.

The important points are as follows:

  • Line 19 to 21 of Listing 29 defines the output data defined at grid cells named “rain”.

  • Listing 30 follows the flow chart in The flowchart of the model.

  • On line 41 of Listing 30, iRICMI_ROut_Grid_Cell_Real is called, to register the variable rain, that stores the value of output “rain” defined in definition.xml. The value of rain is sent to iricmi_server when iRICMI_Model_Sync is called.

Listing 30 source code main.f90 of grid_att_send
 1program grid_att_send
 2	use iricmi
 3	implicit none
 4
 5	integer:: ier
 6	integer:: canceled
 7
 8	double precision:: time
 9	double precision:: dt
10	double precision:: endtime
11
12	integer:: isize, jsize, nodeCount, cellCount
13	double precision, dimension(:), allocatable:: gridX, gridY
14	double precision:: rain_val
15	double precision, dimension(:), allocatable:: rain
16
17	call iRICMI_Model_Init(ier)
18
19	time = 0
20	dt = 1
21	endtime = 100
22
23	! read grid
24	call iRICMI_Read_Grid2d_Str_Size(isize, jsize, ier)
25	nodeCount = isize * jsize
26	cellCount = (isize - 1) * (jsize - 1)
27	allocate(gridX(nodeCount))
28	allocate(gridY(nodeCount))
29	allocate(rain(cellCount))
30	call iRICMI_Read_Grid2d_Coords(gridX, gridY, ier)
31
32	! set rain value
33	rain_val = 0.1
34	rain = rain_val
35
36	! register time
37	call iRICMI_ROut_Time(time, ier)
38	call iRICMI_ROut_Exchange_Interval(dt, ier)
39	call iRICMI_ROut_Dump_Interval(dt, ier)
40
41	! register grid attribute to output
42	call iRICMI_ROut_Grid_Cell_Real("rain", rain, ier)
43
44	! first sync before entering time loop
45	call iRICMI_Model_Sync(ier)
46
47	print "('In grid_att_send t = ', f5.1, ', rain = ', f8.1)", time, rain_val
48
49	do while (1)
50		time = time + dt
51
52		! set rain value
53		rain_val = rain_val + 0.1
54		rain = rain_val
55
56		if (time > endtime) then
57			exit
58		end if
59		
60		call iRICMI_Check_Cancel(canceled)
61		if (canceled == 1) then
62			exit
63		end if
64
65		call iRICMI_Model_Sync(ier)
66
67		print "('In grid_att_send t = ', f5.1, ', rain = ', f8.1)", time, rain_val
68	end do
69
70	call iRICMI_Model_Terminate(ier)
71	stop
72
73end program grid_att_send

grid_att_recieve

Listing 31 shows “definition.xml” of grid_att_receive, and Listing 32 shows the source code of grid_att_receive.

The important points are as follows:

  • Line 19 to 23 of Listing 31 defines the grid attribute “rain” as time dependent value (Refer to Defnining a grid attribute).

  • Listing 32 follows the flow chart in The flowchart of the model.

Project data

The project data to run grid_att_send and grid_att_receive connected is stored in testdata/grid_att_send_receive_fortran folder.

Listing 33 shows iricmi_project.xml. You can see that a connection is defined to exchange data of “rain”.

As explained in Running models in “coupled” mode, you can run the models for the project, by “chdir”ing to the project folder, and launch iricmi_run.

Listing 33 iricmi_project.xml for grid_att_send and grid_att_receive
 1<iRICMIProject>
 2	<Models>
 3		<Model name="grid_att_send" nodes="1" />
 4		<Model name="grid_att_receive" nodes="1" />
 5	</Models>
 6	<Connections>
 7		<Connection>
 8			<From model="grid_att_send" type="gridAttribute" name="rain" />
 9			<To model="grid_att_receive" type="gridAttribute" name="rain" />
10		</Connection>
11	</Connections>
12</iRICMIProject>
