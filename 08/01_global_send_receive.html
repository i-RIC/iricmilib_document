global_send and global_receive

global_send and global_receive are the most basic sample models. They just send and receive time dependent scalar value “discharge”.

global_send and global_receive are implemented in FORTRAN, C++ and Python. On this page FORTRAN version is explained.

global_send

Listing 22 shows “definition.xml” of global_send, and Listing 23 shows the source code of global_send.

The important points are as follows:

  • Line 16 to 18 of Listing 22 defines the global output value “discharge”.

  • Listing 23 follows the flow chart in The flowchart of the model.

  • On line 27 of Listing 23, iRICMI_ROut_Real is called, to register the variable discharge, that stores the value of output “discharge” defined in definition.xml. The value of discharge is sent to iricmi_server when iRICMI_Model_Sync is called.

Listing 22 definition.xml of global_send
 1<SolverDefinition
 2	xmlns="www.iric.net/SolverDefinition/1.0"
 3	name="global_send_fortran"
 4	caption="Model for testing sending global value (FORTRAN)"
 5	version="1.0"
 6	copyright="iRIC organization"
 7	release="2020.6.20"
 8	homepage="http://i-ric.org"
 9	executable="global_send_fortran.exe"
10	iterationtype="time"
11	gridtype="nogrid"
12>
13	<CalculationCondition>
14	</CalculationCondition>
15	<GlobalOutput>
16		<Output name="discharge" caption="Discharge">
17			<Definition valueType="real" />
18		</Output>
19	</GlobalOutput>
20</SolverDefinition>
Listing 23 source code main.f90 of global_send
 1program global_send
 2	use iricmi
 3	implicit none
 4
 5	integer:: ier
 6	integer:: canceled
 7
 8	double precision:: time
 9	double precision:: dt
10	double precision:: endtime
11
12	double precision:: discharge
13
14	call iRICMI_Model_Init(ier)
15
16	time = 0
17	dt = 1
18	endtime = 100
19
20	discharge = 0
21
22	! register time
23	call iRICMI_ROut_Time(time, ier)
24	call iRICMI_ROut_Exchange_Interval(dt, ier)
25
26	! register variable to output value to other models
27	call iRICMI_ROut_Real("discharge", discharge, ier)
28
29	! first sync before entering time loop
30	call iRICMI_Model_Sync(ier)
31
32	print "('In global_send t = ', f5.1, ', discharge = ', f8.1)", time, discharge
33
34	do while (1)
35		time = time + dt
36		discharge = discharge + 0.1;
37
38		if (time > endtime) then
39			exit
40		end if
41		
42		call iRICMI_Check_Cancel(canceled)
43		if (canceled == 1) then
44			exit
45		end if
46		
47		call iRICMI_Model_Sync(ier)
48
49		print "('In global_send t = ', f5.1, ', discharge = ', f8.1)", time, discharge
50	end do
51
52	call iRICMI_Model_Terminate(ier)
53	stop
54end program

global_recieve

Listing 24 shows “definition.xml” of global_receive, and Listing 25 shows the source code of global_receive.

The important points are as follows:

  • Line 15 to 20 of Listing 24 defines the calculation condition “discharge” as time dependent value (Refer to Defining a scalar value).

  • Listing 25 follows the flow chart in The flowchart of the model.

  • On line 25 of Listing 25, iRICMI_RIn_Real is called, to register the variable discharge, that stores the value of time dependent input “discharge” defined in definition.xml. The value of discharge is received from iricmi_server when iRICMI_Model_Sync is called.

Project data

The project data to run global_send and global_receive connected is stored in testdata/global_send_receive_fortran folder.

Listing 26 shows iricmi_project.xml. You can see that a connection is defined to exchange data of “discharge”.

As explained in Running models in “coupled” mode, you can run the models for the project, by “chdir”ing to the project folder, and launch iricmi_run.

Listing 26 iricmi_project.xml for global_send and global_receive
 1<iRICMIProject>
 2	<Models>
 3		<Model name="global_send" nodes="1" />
 4		<Model name="global_receive" nodes="1" />
 5	</Models>
 6	<Connections>
 7		<Connection>
 8			<From model="global_send" type="global" name="discharge" />
 9			<To model="global_receive" type="calculationCondition" name="discharge" />
10		</Connection>
11	</Connections>
12</iRICMIProject>
