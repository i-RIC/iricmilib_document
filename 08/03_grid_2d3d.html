grid2d3d

grid2d3d is a model that reads two dimensional grid, generate three dimensional grid inside, and send values defined at grid nodes, cells, and edges of two dimensional and three dimensional grids. It also receive values defined at grid nodes, cells, and edges.

Also it is possible to change the grid coordinates at every time step, by giving dx and dy values other than zero.

In the sample project, two regions are defined for model grid2d3d, and the values are exchanged in both direction.

grid2d3d is implemented in FORTRAN, C++ and Python. On this page FORTRAN version is explained.

grid2d3d

Listing 36 shows “definition.xml” of grid2d3d, and Listing 37 shows the source code of grid2d3d.

The important points are as follows:

  • In Listing 36, two <GridType> element is defined. The first one with name”input” is for two dimensional grid, and the other one with name “output3d” is for three dimensional grid. It is important that the for <GridType> with name “output3d”, attribute “post” is defined, and the value is “true”. It means that the grid should not be created in Pre-processing window, but generated inside the model, and used only for post-processing.

  • Output data is defined for both two dimensional grid and three dimensional grid. For two dimensional grid, the names are “outputNode2d”, “outputCell2d”, etc. For three dimensional grid, the names are “outputNode3d”, “outputCell3d”, etc.

  • Time dependent input data is defined for both two dimensional grid and three dimensional grid. For two dimensional grid, the names are “inputNode2d”, “inputCell2d”, etc. For three dimensional grid, the names are “inputNode3d”, “inputCell3d”, etc.

  • Line 95 of Listing 37 calls iRICMI_RIn_Grid3d_Coords_WithGridId, to output the grid. The three dimensional grid is saved to Case1.cgn, and also sent to iricmi_server. It is important that the value of the first argument is “output3d”, that matches the name of grid type defined in Listing 36.

  • Line 103 and 104 of Listing 37 calls iRICMI_ROut_Grid2d_Coords_WithGridId and iRICMI_ROut_Grid3d_Coords_WithGridId. Calling these subroutines, the model passes the variables that stores the coordinates of the grid, and informs that the shape of the grids can change.

Listing 37 source code main.f90 of grid2d3d
  1subroutine updateData(t, isize, jsize, ksize, data)
  2	double precision, intent(in):: t
  3	integer, intent(in):: isize
  4	integer, intent(in):: jsize
  5	integer, intent(in):: ksize
  6	double precision, dimension(isize, jsize, ksize), intent(inout):: data
  7	integer:: i, j, k
  8	double precision:: v
  9
 10	do k = 1, ksize
 11		do j = 1, jsize
 12			do i = 1, isize
 13				v = sin(t * 0.05 + (i + j + k - 3) * 0.1)
 14				data(i, j, k) = v
 15			end do
 16		end do
 17	end do
 18end subroutine

 20program grid2d3d
 21	use iricmi
 22	implicit none
 23
 24	integer:: ier
 25	integer:: canceled
 26
 27	double precision:: dx, dy
 28	integer:: i, j, k
 29
 30	double precision:: time
 31	double precision:: dt
 32	double precision:: endtime
 33
 34	integer:: isize, jsize, ksize
 35	double precision, dimension(:, :), allocatable:: gridX, gridY
 36	double precision, dimension(:, :, :), allocatable:: grid3dX, grid3dY, grid3dZ
 37	
 38	double precision, dimension(:, :, :), allocatable:: outputNode2d, outputCell2d, outputIFace2d, outputJFace2d

 45
 46	time = 0
 47	dt = 1
 48	endtime = 100
 49
 50	! read grid
 51	call iRICMI_Read_Grid2d_Str_Size(isize, jsize, ier)
 52	allocate(gridX(isize, jsize), gridY(isize, jsize))
 53	allocate(outputNode2d(isize, jsize, 1))
 54	allocate(outputCell2d(isize - 1, jsize - 1, 1))
 55	allocate(outputIFace2d(isize, jsize - 1, 1))
 56	allocate(outputJFace2d(isize - 1, jsize, 1))
 57	allocate(inputNode2d(isize, jsize, 1))
 58	allocate(inputCell2d(isize - 1, jsize - 1, 1))

 70	allocate(outputCell3d(isize - 1, jsize - 1, ksize - 1))
 71	allocate(outputIFace3d(isize, jsize - 1, ksize - 1))
 72	allocate(outputJFace3d(isize - 1, jsize, ksize - 1))
 73	allocate(outputKFace3d(isize - 1, jsize - 1, ksize))
 74	allocate(inputNode3d(isize, jsize, ksize))
 75	allocate(inputCell3d(isize - 1, jsize - 1, ksize - 1))
 76	allocate(inputIFace3d(isize, jsize - 1, ksize - 1))
 77	allocate(inputJFace3d(isize - 1, jsize, ksize - 1))
 78	allocate(inputKFace3d(isize - 1, jsize - 1, ksize))

 82			grid3dX(i, j, :) = gridX(i, j)
 83			grid3dY(i, j, :) = gridY(i, j)
 84		end do
 85	end do
 86	do k = 1, ksize
 87		grid3dZ(:, :, k) = k - 1
 88	end do
 89
 90	call iRICMI_Write_NamedGrid3d_Coords_WithGridId("output3d", isize, jsize, ksize, grid3dX, grid3dY, grid3dZ, grid3did, ier)

101	! register grid attribute to output
102	call iRICMI_ROut_Grid_Node_Real_WithGridId(1, "outputNode2d", outputNode2d, ier)

107	call iRICMI_ROut_Grid_Node_Real_WithGridId(1, "receivedNode2d", inputNode2d, ier)
108	call iRICMI_ROut_Grid_Cell_Real_WithGridId(1, "receivedCell2d", inputCell2d, ier)
109	call iRICMI_ROut_Grid_IFace_Real_WithGridId(1, "receivedIFace2d", inputIFace2d, ier)
110	call iRICMI_ROut_Grid_JFace_Real_WithGridId(1, "receivedJFace2d", inputJFace2d, ier)
111
112	call iRICMI_RIn_Grid_Node_Real_WithGridId(1, "inputNode2d", inputNode2d, ier)
113	call iRICMI_RIn_Grid_Cell_Real_WithGridId(1, "inputCell2d", inputCell2d, ier)
114	call iRICMI_RIn_Grid_IFace_Real_WithGridId(1, "inputIFace2d", inputIFace2d, ier)
115	call iRICMI_RIn_Grid_JFace_Real_WithGridId(1, "inputJFace2d", inputJFace2d, ier)
116
117	call iRICMI_ROut_Grid_Node_Real_WithGridId(grid3did, "outputNode3d", outputNode3d, ier)
118	call iRICMI_ROut_Grid_Cell_Real_WithGridId(grid3did, "outputCell3d", outputCell3d, ier)
119	call iRICMI_ROut_Grid_IFace_Real_WithGridId(grid3did, "outputIFace3d", outputIFace3d, ier)
120	call iRICMI_ROut_Grid_JFace_Real_WithGridId(grid3did, "outputJFace3d", outputJFace3d, ier)
121	call iRICMI_ROut_Grid_KFace_Real_WithGridId(grid3did, "outputKFace3d", outputKFace3d, ier)
122
123	call iRICMI_ROut_Grid_Node_Real_WithGridId(grid3did, "receivedNode3d", inputNode3d, ier)
124	call iRICMI_ROut_Grid_Cell_Real_WithGridId(grid3did, "receivedCell3d", inputCell3d, ier)
125	call iRICMI_ROut_Grid_IFace_Real_WithGridId(grid3did, "receivedIFace3d", inputIFace3d, ier)
126	call iRICMI_ROut_Grid_JFace_Real_WithGridId(grid3did, "receivedJFace3d", inputJFace3d, ier)
127	call iRICMI_ROut_Grid_KFace_Real_WithGridId(grid3did, "receivedKFace3d", inputKFace3d, ier)
128
129	call iRICMI_RIn_Grid_Node_Real_WithGridId(grid3did, "inputNode3d", inputNode3d, ier)
130	call iRICMI_RIn_Grid_Cell_Real_WithGridId(grid3did, "inputCell3d", inputCell3d, ier)
131	call iRICMI_RIn_Grid_IFace_Real_WithGridId(grid3did, "inputIFace3d", inputIFace3d, ier)
132	call iRICMI_RIn_Grid_JFace_Real_WithGridId(grid3did, "inputJFace3d", inputJFace3d, ier)
133	call iRICMI_RIn_Grid_KFace_Real_WithGridId(grid3did, "inputKFace3d", inputKFace3d, ier)
134
135	call updateData(time, isize, jsize, 1, outputNode2d)
136	call updateData(time, isize - 1, jsize - 1, 1, outputCell2d)
137	call updateData(time, isize, jsize - 1, 1, outputIFace2d)
138	call updateData(time, isize - 1, jsize, 1, outputJFace2d)
139
140	call updateData(time, isize, jsize, ksize, outputNode3d)
141	call updateData(time, isize - 1, jsize - 1, ksize - 1, outputCell3d)
142	call updateData(time, isize, jsize - 1, ksize - 1, outputIFace3d)
143	call updateData(time, isize - 1, jsize, ksize - 1, outputJFace3d)
144	call updateData(time, isize - 1, jsize - 1, ksize, outputKFace3d)
145
146	! first sync before entering time loop
147	call iRICMI_Model_Sync(ier)
148
149	print "('In grid2d3d t = ', f5.1)", time
150
151	do while (.true.)
152		time = time + dt
153
154		if (time > endtime) then
155			exit
156		end if
157		
158		call iRICMI_Check_Cancel(canceled)
159		if (canceled == 1) then
160			exit
161		end if
162
163		gridX(:, :) = gridX(:, :) + dx
164		gridY(:, :) = gridY(:, :) + dy
165
166		grid3dX(:, :, :) = grid3dX(:, :, :) + dx
167		grid3dY(:, :, :) = grid3dY(:, :, :) + dy
168
169		call updateData(time, isize, jsize, 1, outputNode2d)
170		call updateData(time, isize - 1, jsize - 1, 1, outputCell2d)
171		call updateData(time, isize, jsize - 1, 1, outputIFace2d)
172		call updateData(time, isize - 1, jsize, 1, outputJFace2d)
173
174		call updateData(time, isize, jsize, ksize, outputNode3d)
175		call updateData(time, isize - 1, jsize - 1, ksize - 1, outputCell3d)
176		call updateData(time, isize, jsize - 1, ksize - 1, outputIFace3d)
177		call updateData(time, isize - 1, jsize, ksize - 1, outputJFace3d)
178		call updateData(time, isize - 1, jsize - 1, ksize, outputKFace3d)
179
180		call iRICMI_Model_Sync(ier)
181
182		print "('In grid2d3d t = ', f5.1)", time
183	end do
184
185	call iRICMI_Model_Terminate(ier)
186	stop
187end program

Project data

The project data to run grid2d3d is stored in testdata/grid2d3d_fortran folder.

Listing 38 shows iricmi_project.xml. You can see that connections are defined to exchange data between two regions, for both two dimensional and three dimensional grids.

Fig. 6 and Fig. 7 shows the shape of grids defined for “region1” and “region2”. The gray area in Fig. 7 corresponds to the grid region of “region1”, so the grid of “region2” is inside that of “region1”.

For “region2”, dx = 0.01, dy = -0.02. It means that for “region2” the region of the grids slide every time step.

As explained in Running models in “coupled” mode, you can run the models for the project, by “chdir”ing to the project folder, and launch iricmi_run.
